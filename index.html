What is this?

Paste the raw JSON exported from a Kibana alert. The tool will automatically extract key fields like rule name, host, user, IPs, and timestamps.

How to get it

  • Open the alert in Kibana
  • Click the JSON tab on the alert detail view
  • Click Copy to clipboard (top right)
  • Paste it into the box below

The event timestamp will be auto-injected into your timeline as ALERT TRIGGERED (for alerts) or EVENT LOGGED (for plain logs).

What to log

Build a chronological picture of everything that happened around the alert. Pull events from:

  • The original alert timestamp (auto-added)
  • Related process creation events (Event ID 4688)
  • Network connection logs
  • Authentication events (logons, failures)
  • Any prior or subsequent alerts on the same host/user
  • When you began and completed your investigation

📖 Investigation Guide

If this alert has an investigation guide, it will appear below — click Open in new tab to view full triage steps, detection logic and remediation actions specific to this rule.

What to cover in your analysis

  • What exactly triggered the alert and why?
  • Is the behaviour expected for this user/host/role?
  • What do external tools show? (VirusTotal, Shodan, threat intel)
  • Are there related alerts or historical matches?
  • What is the kill chain stage if malicious?
Example
The alert fired on a PowerShell execution using -EncodedCommand and -ExecutionPolicy Bypass flags. The encoded command decodes to a download cradle fetching a script from 192.168.1.100. VirusTotal shows 0/72 detections for the IP however it does not appear in any threat intel feeds. The parent process cmd.exe is unusual for this user — jsmith is a finance analyst with no legitimate reason to run encoded PowerShell. No prior alerts on this host in the last 30 days. The destination port 4444 is commonly associated with Metasploit listeners.

🏁Summary & Conclusion ▼
💡 Select the verdict that best describes the outcome of your investigation.help ▾

Verdicts explained

  • False Positive — Alert fired incorrectly. Rule needs tuning.
  • Benign — Activity is confirmed legitimate. No threat present.
  • True Positive — Confirmed malicious or policy-violating activity.
💡 Select the category that best describes this incident.help ▾

Activity type definitions

  • Malware — Virus, ransomware, trojan, RAT, etc.
  • Hacking — Exploitation, C2, lateral movement, privilege escalation
  • Social — Phishing, pretexting, BEC
  • Misuse — Policy violation, insider threat
  • Physical — Unauthorised physical access
  • Error — Misconfiguration, accidental action
💡 Describe the confirmed or likely impact of this incident on operations, data, and reputation.help ▾

What to write

Describe the confirmed or likely damage caused by this incident. Consider:

  • Confidentiality — was data exfiltrated or accessed without authorisation?
  • Integrity — were systems or data modified?
  • Availability — were services disrupted?
  • Scope — is this isolated to one host or has it spread?
Example
A reverse shell was established on DESKTOP-GH7K21 under user jsmith. Lateral movement across the network is possible, with risk of credential harvesting and exfiltration of sensitive finance data. Impact is rated HIGH.
💡 Write a clear summary of your findings and the reason for your verdict.help ▾

What to write

Write 2–4 sentences that someone unfamiliar with the case can read and immediately understand. Include:

  • What happened in plain English
  • Why you reached your verdict
  • Any mitigating or aggravating factors
Example — True Positive
A PowerShell download cradle was executed on DESKTOP-GH7K21 by finance analyst jsmith, spawned from cmd.exe with execution policy bypass flags. The encoded command retrieves a script from an internal IP on port 4444, consistent with C2 activity. The behaviour is anomalous for this user and host. Escalated to Incident Response for containment.
Example — False Positive
The encoded PowerShell command was confirmed as part of the organisation's SCCM software deployment process. The parent process and flags match the known deployment template. Rule has been updated to suppress alerts from the SCCM service account.
